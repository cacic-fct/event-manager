---
title: Formulários
---

A tela de formulários serve para criar perguntas, vincular formulários a eventos ou grandes eventos, publicar respostas e acompanhar resultados.

Use formulários para dados complementares de inscrição, avaliações, confirmações, enquetes simples ou coletas pós-evento. Não use formulários para substituir fluxos já existentes, como inscrição, presença, pagamento, certificado, etc.

## Criar formulário

Ao criar um formulário, defina:

- Nome e descrição;
- Evento ou grande evento dono;
- Sigilo;
- Modo de contagem de respostas;
- Campos do formulário;
- Vínculos onde ele aparecerá.

O dono organiza o formulário no painel. Os vínculos definem onde a pessoa pode responder.

## Sigilo

Escolha o sigilo antes de divulgar o formulário.

| Sigilo | Quando usar |
| --- | --- |
| Público | Quando identidade e respostas podem aparecer para pessoas autorizadas. |
| Parcialmente sigiloso | Quando administradores precisam ver respostas, mas outros visualizadores não devem ver respostas individuais. |
| Sigiloso | Quando apenas administradores devem ver identidade e respostas individuais. |
| Anônimo | Quando resultados não devem expor identidade para terceiros. |

Não prometa anonimato fora do que o sistema mostra. Mesmo em formulários anônimos, o sistema pode preservar rastreabilidade técnica mínima para auditoria e proteção contra abuso.

## Modo de respostas

| Modo | Efeito |
| --- | --- |
| Uma resposta por evento | A pessoa responde uma vez para cada evento ou para cada grande evento vinculado. |
| Várias respostas por evento | A pessoa pode enviar mais de uma resposta no mesmo alvo. |
| Uma resposta para o formulário inteiro | A primeira resposta vale para todos os vínculos do formulário. |

Use "uma resposta para o formulário inteiro" apenas quando a resposta realmente for relevante para ser compartilhada entre todos os eventos onde o formulário aparece.

## Vínculos

Um formulário só aparece para o público quando possui vínculo com evento ou grande evento.

Em cada vínculo, revise:

- Quem pode responder: inscritos, participantes com presença ou ambos;
- Data de início e fim;
- Se o formulário entra na inscrição;
- Se a resposta é obrigatória para concluir a inscrição;
- Se perguntas obrigatórias serão validadas;
- Se pessoas elegíveis serão notificadas;
- Se ministrantes podem publicar manualmente, quando aplicável.

Formulários inseridos na inscrição devem ser curtos e claramente necessários.  
Um formulário obrigatório não preenchido impede a conclusão da inscrição.

## Publicação

**Salvar não publica automaticamente.**

Use:

- **Rascunho** para guardar uma versão sem publicar;
- **Salvar** para gravar alterações administrativas;
- **Agendar** para publicar em uma data futura;
- **Publicar** para disponibilizar imediatamente;
- **Despublicar** para tirar do ar.

Depois de publicar, confira o formulário pelo app público quando possível.

## Resultados

A aba de resultados mostra resumo e respostas conforme o sigilo e as permissões do usuário.

O botão **CSV** exporta respostas quando a permissão estiver disponível. Trate o arquivo exportado como dado pessoal quando ele contiver identidade, e-mail, respostas livres ou outros identificadores.

## Resultados públicos

Marque **Publicar resultados para público autorizado** apenas quando as pessoas que responderam puderem ver o resumo.

Marque **Resultados ao vivo** apenas quando for aceitável que o resumo das respostas seja disponibilizado com o formulário ainda aberto.  
Sem resultados ao vivo, os resultados públicos aparecem somente quando o formulário encerra.

## Exclusão

Excluir um formulário com respostas pode apagar contexto operacional importante. Dessa forma, prefira despublicar ou desativar o formulário ao invés de excluir.

Antes de excluir, confira:

- Se há respostas;
- Se o formulário foi usado em inscrição;
- Se os resultados foram usados para decisão do evento;
- Se há obrigação de retenção para auditoria.
